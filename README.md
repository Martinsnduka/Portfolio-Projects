# 🛒 E-Commerce Data Exploration & Analysis — SQL Project 


##  Project Overview

This project explores a real-world E-commerce dataset using **SQL** to surface actionable insights on customer behaviour, order trends, and product performance. The analysis is structured in two layers 
1. Exploratory data analysis
2. Advanced EDA to answer business questions that drive decisions in revenue,and marketing

##  Business Objective

Retail and e-commerce businesses generate enormous volumes of transactional data daily. This project demonstrates how **structured SQL querying** can transform raw data into clear business intelligence, answering the questions that matter most to stakeholders.


##  Dataset

The dataset captures end-to-end e-commerce transactions across three related tables : Products,Fact_sales,and customers

**Tool:** Microsoft SQL Server Management Studio (SSMS)

## Key Business Questions Answered

### E-commerce Data Exploration.sql

 1.  Which are the **top 5 best-performing products** by total revenue generated? 
 2. Which products are **worst-performing** in terms of sales volume? 
 3. Who are the **top customers** ranked by total order value? |

### Advanced_SQL_EDA.sql
 4. How has **sales performance trended over time**? |
 5. How do **yearly product sales** compare against both the product's average and the prior year? 
 6. Which **product categories** contribute the highest percentage of total sales? 
 7. How are **products distributed across cost segments** , and how many products fall into each tier? 


## SQL Techniques Demonstrated

- Aggregations (`SUM`, `COUNT`, `AVG`)
- Window functions (`RANK()`, `LAG()`, `PARTITION BY`)
- Common Table Expressions (CTEs)
- Subqueries and derived tables
- `CASE WHEN` logic for segmentation
- Year-over-year comparisons using `LAG()`
- Percentage of total using window aggregation


## How to Run

1. Open **SQL Server Management Studio (SSMS)**
2. Connect to your local or remote SQL Server instance
3. Import or restore the e-commerce dataset into your database
4. Open either `.sql` file and execute queries individually or in full
5. Results will return directly in the SSMS results pane
